**Date saved**: 2026-05-25

This folder preserves the external research inputs used to revise VisuTry's B2B commerce commercialization roadmap.

## Files

- `best-commercialization-path-ai-eyewear-vto-2026.pdf`
  - Broad market research on the best commercialization path for an AI eyewear virtual try-on product in 2026.
- `ai-eyewear-vto-commercialization-gemini.pdf`
  - Gemini research report on AI eyewear try-on commercialization paths, competitors, pricing, customer segments, and 90-day execution.

## Decision Link

The conclusions from these reports were incorporated into:

- `docs/strategy/2026-05-25-b2b-commerce-commercialization-roadmap.md`

## Key Takeaways Adopted

- Do not keep VisuTry primarily as a standalone B2C credits tool.
- Make merchant-facing virtual try-on the main commercialization path.
- Prioritize a generic widget and merchant design-partner package.
- Treat Shopify as a high-priority beta/distribution shell, not a separate core product.
- Keep WooCommerce second and BigCommerce later.
- Use agency/co-brand distribution selectively, especially through boutique ecommerce agencies.
- Build one reusable backend and distribute it through multiple shells.
- Treat AI-assisted development as an accelerator, not a substitute for security, privacy, performance, merchant trust, and support quality.

## Gemini Follow-Up Revision

After the Gemini report, an additional revision was provided:

- AI-assisted development reduces the cost and time needed to produce widget and platform wrappers.
- Therefore Shopify/WooCommerce beta work can run in parallel with the generic widget, as long as it does not replace merchant validation.
- The durable moat should shift from code to distribution, workflow depth, vertical trust, ROI analytics, and harder optical workflows such as PD research, pre-shop, walk-out recovery, and EHR/PMS discovery.
- AI-generated code in merchant, auth, billing, privacy, or multi-tenant paths still requires human review and security testing.
